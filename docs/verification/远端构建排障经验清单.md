# 远端构建排障经验清单（rsync + thirdparty + build.sh）

本文沉淀一次实际远端构建过程中的高频问题，适用于通过 `scripts/build/remote_build_run_datasystem.sh` 在远端执行同步、构建、测试的场景。

---

## 1. 先判断问题在哪一层

按顺序分层判断，避免混在一起排：

1. **连接层**：`ssh` 能否连上、主机别名是否可解析。  
2. **同步层**：`rsync` 是否完成，是否误忽略源码目录。  
3. **依赖层**：系统工具链是否齐全（`patch/autoconf/automake/libtool/perl` 等）。  
4. **构建层**：`build.sh` 是否能进入主工程构建（不止 thirdparty）。  
5. **测试层**：`ctest`、`validate_kv_executor.sh`、`run_example` 是否通过。

---

## 2. 本次真实故障与结论

### 2.1 thirdparty patch 文件缺失

- **现象**：日志出现  
  - `file SHA256 failed to read file ... third_party/patches/... No such file or directory`
- **根因**：`rsync ignore` 误把 `third_party/`（源码目录）排除了。
- **处理**：只忽略 `.third_party/`（缓存目录），不要忽略 `third_party/`。
- **建议规则**：保留
  - `.third_party/`
  - `*/.third_party/`
  - `.venv/`
  - `build_cov/`

### 2.2 工具链命令缺失

- **现象**：日志出现  
  - `patch executable not found!`  
  - `autoconf: command not found`  
  - `You must have automake/libtool installed`
- **根因**：远端基础构建工具未安装完整。
- **处理（CentOS Stream 9）**：

```bash
sudo dnf install -y patch autoconf automake libtool perl-FindBin
```

### 2.3 OpenSSL thirdparty 配置失败

- **现象**：日志出现  
  - `Can't locate FindBin.pm ... Configure line 15`
- **根因**：Perl 模块 `FindBin` 缺失。
- **处理**：

```bash
sudo dnf install -y perl-FindBin
perl -MFindBin -e 'print qq(ok\n)'
```

### 2.4 Ascend 依赖缺失

- **现象**：日志出现  
  - `Could NOT find Ascend (missing: ASCEND_LIBRARY HCCL_LIBRARY ASCEND_INCLUDE_DIR)`
- **根因**：机器未安装 Ascend/HCCL SDK（或环境变量未配）。
- **处理**：按需求二选一  
  - 需要 Ascend：安装对应 SDK 并配置路径。  
  - 不需要 Ascend：在工程构建参数中关闭 Ascend 相关开关（以项目 CMake 选项为准）。

### 2.5 变量展开导致缓存路径异常

- **现象**：日志路径出现字面量 `$HOME`，例如 `.../$HOME/.cache/...`
- **根因**：`DS_OPENSOURCE_DIR` 通过脚本传递时未正确在远端展开。
- **处理**：传参时使用远端实际绝对路径（如 `/root/.cache/...`），或在远端 shell 中先展开再导出。

## 3. 远端构建实时观察命令

### 3.1 看任务是否还在跑

```bash
ssh <remote> "ps -eo pid,ppid,cmd,%cpu,%mem --sort=-%cpu | sed -n '1,20p'"
```

### 3.2 快速看 thirdparty 失败点

```bash
ssh <remote> "cd <DS>/build/multi_build && for d in *; do test -f \"$d/log\" && echo \"===== $d =====\" && rg -n \"error|failed|not found|No such file\" \"$d/log\" -i -m 3; done"
```

### 3.3 针对单个库看完整日志

```bash
ssh <remote> "sed -n '1,160p' <DS>/build/multi_build/openssl/log"
ssh <remote> "sed -n '1,160p' <DS>/build/multi_build/absl/log"
```

## 4. 预防清单（每次开跑前）

```bash
# 1) 连通与身份
ssh <remote> "whoami && hostname"

# 2) 构建基础依赖
ssh <remote> "command -v gcc g++ cmake make patch autoconf automake libtool perl"
ssh <remote> "perl -MFindBin -e 'print qq(ok\n)'"

# 3) rsync 仅 dry-run
rsync -az --delete --dry-run --exclude-from=<ignore-file> <local_ds>/ <remote>:<remote_ds>/

# 4) 并行参数
# 16C 机器建议 build=16, ctest=16 起步；失败后再降并发观察是否资源竞争。
```

---

## 5. 是否需要网页界面

结论：**短期不必先做完整 Web 平台，先做“日志可视化最小页”即可**。

- 立即可用方案：先把远端构建日志落盘到固定目录，使用 `python3 -m http.server` 或轻量 Nginx 只读展示。
- 若多人协作再升级：做一个小页面，展示  
  - 任务状态（sync/build/ctest）  
  - 最近失败库与错误摘要  
  - 日志下载链接  
  - 触发命令与参数（含 `--jobs`、`--ctest-jobs`）
- 不建议第一步就做复杂调度系统；先保证“可看、可查、可复现”。
