# datasystem UB/URMA 压测用例

## 测试目标

| 目标 | 说明 |
|------|------|
| **性能目标** | P99 < 2 ms，吞吐达标 |
| **可靠性目标** | QPS 和并发度大幅超过负荷不崩溃，回落后功能继续可用 |

## 规格参数

**数据大小**：0.5 MB / 8 MB（精排最大）/ 12 MB（召排最大）

**变量**：QPS 尖峰倍数 = 尖峰 / Avg QPS

**环境 Setup**：

| 组件 | 规格 | 说明 |
|------|------|------|
| kv worker | 召排 8C192G，精排 8C16G | 固定 1 个节点 1 个 |
| kv client | 8C32G | Predicator 主图/子图 |

**kv worker 读写视角**：

| 模式 | QPS | 说明 |
|------|-----|------|
| 共享内存访问 | 35 × K | kv worker 触发 Get → 共享内存读 |
| 入口带宽占用 | 35 × K | kv worker 触发远端数据拉取 Get → 入口带宽 |
| 基于共享内存 Copy | 35 × K | kv worker 本地接收 Put → 跨 NUMA HCCS 带宽 |
| 出口带宽占用 | 35 × K | 其他 kv worker 跨级拉取，本 worker 提供数据的处理 QPS |

**核心生命周期管理**：

- 召排：缓存淘汰（整体数据容量 10% / 50% / 100%）
- 精排：TTL（5s）+ 缓存淘汰

**验收标准**：成功率 100%，P99 < 2 ms

## 操作说明

### P1：召排基线性能

```
目的：建立 datasystem UB/URMA 环境下 12MB 对象的性能基线

前置：1 Worker + etcd，enable_urma=true，8C32G Client

1. 压测工具设置：并发 10，QPS 350，写入 key="召回_$i"，12 MB 对象，≥10000 条
2. 每 10s 抓取 Metrics Summary：grep 'Compare with' worker.INFO.log | tail -3
3. 记录 client_rpc_get_latency max、worker_urma_read_latency max
4. 确认 urma 传输：client_*_urma_*_bytes delta > 0
5. 验收：P99 < 2ms，成功率 100%
```

### S3：召排极限可靠性（K=10）

```
目的：验证 QPS 尖峰达到设计上限 10 倍时系统不崩溃，流量回落后继续可用

前置：同 P1

1. 压测：并发 100，QPS 3500，持续压测 60s
2. 监控 resource.log：SHARED_MEMORY、ETCD_QUEUE、WAITING_TASK_NUM
3. 检查 Worker 日志：有无 OOM、Worker 退出、code=5/6/7
4. 压测结束后立即恢复低压（QPS 350），验证功能恢复
5. 验收：压测中成功率 ≥ 99%，Worker 未崩溃，回落后成功率恢复 100%
```

### L1：召排缓存淘汰（10%容量）

```
目的：验证容量达到 10% 上限时 LRU 淘汰正确执行

前置：同 P1，Worker 设置 SHM 上限使 10% 容量约能存 1000 条

1. 先写入 10000 条 12MB 数据填满缓存（触发淘汰）
2. 记录前 1000 条的 key 列表
3. 继续写入新数据 5000 条
4. 读取第 1 步中记录的旧 key（已被淘汰）
5. 验收：旧 key Get 返回 NOT_FOUND（被 LRU 淘汰），新 key 正常读写成功率 100%
```

### L4：精排 TTL 过期（5s）

```
目的：验证精排数据 TTL 5s 后正确过期

前置：同 P1，无容量限制

1. 写入 key="精排_$i"，8 MB，等待 5s 以上（可设 10s）
2. 读取该 key
3. 验收：Get 返回 NOT_FOUND 或 code=3，日志有 TTL 删除记录
```
