---

## 六、根因分析

### 6.1 Out of memory 错误

```
Shared memory no space in arena: 2
Error while allocating memory
```

**根因**: 192.168.219.108 节点的共享内存耗尽，无法分配新内存。

### 6.2 QueryMeta 高延迟

- 正常 QM: 2-4ms
- 异常 QM: 7-11ms
- **延迟增加 3-5 倍**

**可能原因**:
1. Master 节点负载高
2. 跨网段网络延迟
3. Master 节点资源竞争

### 6.3 PG 高延迟

- 正常 PG: 4-6ms
- 异常 PG: 11-19ms
- **延迟增加 3-4 倍**

**可能原因**:
1. worker2 处理延迟
2. 数据传输延迟
3. RPC 重试

---

## 七、建议

1. **调查 192.168.219.108 节点的内存问题**
   - 为什么共享内存耗尽
   - 是否需要增加内存或调整内存分配策略

2. **调查 192.168.219.108 节点的负载**
   - 作为 Master 和 Worker 双重角色
   - 可能存在资源竞争

3. **增加 QueryMeta 延迟告警**
   - 当前阈值可能过高
   - 建议阈值：5ms

4. **增加内存告警**
   - Out of memory 错误不应该发生
   - 建议增加内存使用率监控
